# Phone Barcode Scanner

Превратите смартфон в беспроводной сканер штрих‑кодов. Приложение для Windows принимает штрих‑коды и QR‑коды со смартфона и передаёт их в любую программу на компьютере — так, будто сработал физический сканер, подключённый по USB.

## Релизы
[Список релизов](https://github.com/savchenkoea/PhoneBarcodeScanner/releases).

## Поддерживаемые платформы

✔ Windows x86  
✔ Windows x64  
❌ Linux

## Как это работает

1. Установите приложение на компьютер и мобильное приложение на смартфон.
2. Запустите сервер — на экране появится QR‑код с параметрами подключения.
3. Отсканируйте QR‑код в мобильном приложении — смартфон подключится к компьютеру.
4. Наведите камеру телефона на штрих‑код или QR‑код.
5. Код мгновенно появится в активной программе на компьютере — в том месте, где стоит курсор.

Программа (1С, Excel, кассовая программа, товароучётная система и любая другая) получает данные как от обычного подключённого сканера: никаких дополнительных настроек в ней не требуется.

## Возможности

- Эмулирует ввод с физического USB‑сканера штрих‑кодов — совместим с любым ПО.
- Поддерживает подключение нескольких смартфонов одновременно.
- Работает по локальной сети Wi‑Fi
- Защищённое соединение: трафик шифруется по TLS, авторизация по одноразовому ключу.
- Настройка префикса и суффиксов к каждому коду (например, добавить Enter после значения).
- Настройки сохраняются в реестре Windows и применяются при следующем запуске.
- Сворачивается в системный трей.

## Требования

- Windows 7 или новее (x86 или x64)
- Смартфон с мобильным приложением Barcode2PC (Android)
- Локальная сеть Wi‑Fi для связи между устройствами

## Установка и запуск

1. Скачайте `PhoneBarcodeScanner_win_x64.exe` (или `x86` для 32‑битных систем).
2. Запустите файл — установка не требуется.
3. При первом запуске откроется диалог настроек.

## Настройка

Откройте диалог настроек кнопкой **«Настройка»** (сервер при этом должен быть остановлен).

| Параметр | Описание | По умолчанию |
|---|---|---|
| IP‑адрес | Сетевой интерфейс, на котором слушает сервер | Первый активный |
| Порт | TCP‑порт сервера | 10001 |
| Префикс | Символ, добавляемый перед каждым кодом | Нет |
| Суффикс 1 | Символ, добавляемый после кода | Enter (CR, 13) |
| Суффикс 2 | Второй символ после кода | Нет |

В списках символов доступны все ASCII‑коды 0–127. Часто используемые значения суффикса: `13 (CR)` — переход на новую строку / подтверждение, `9 (TAB)` — переход к следующему полю.

## Безопасность соединения

Соединение между смартфоном и компьютером защищено на двух уровнях:

**TLS‑шифрование.** При каждом запуске сервера автоматически генерируется самоподписанный RSA‑2048 сертификат. Все данные передаются в зашифрованном виде.

**Авторизация по одноразовому ключу (passkey).** QR‑код на экране содержит одноразовый ключ доступа. После успешного подключения смартфона ключ меняется — повторное использование невозможно. Неавторизованные клиенты немедленно отключаются.

**Certificate pinning.** Мобильное приложение проверяет подлинность сертификата сервера по SHA‑256 отпечатку открытого ключа (формат OkHttp: `sha256/...`). Это защищает от атак типа «человек посередине» даже при самоподписанном сертификате.

## Работа через интернет

Приложение поддерживает только работу в локальной сети Wi‑Fi. Подключение через интернет технически невозможно по следующим причинам:

**Первичное подключение требует физического присутствия.** Для авторизации смартфон должен отсканировать QR‑код, отображаемый на экране компьютера. Сделать это удалённо невозможно.

**Параметры подключения нельзя сохранить.** QR‑код содержит одноразовый ключ авторизации (passkey), который меняется при каждом подключении. Сохранённые данные быстро устареют — при следующем запуске приложения на смартфоне потребуется повторное сканирование нового QR‑кода.

**Сертификат пересоздаётся при каждом запуске.** Поскольку сертификат генерируется заново при каждом старте сервера, мобильное приложение не сможет подтвердить подлинность сервера без повторного сканирования QR‑кода.

## Протокол связи

Смартфон подключается к серверу по **WebSocket over TLS** и обменивается JSON‑сообщениями:

```json
{"command": "scan", "data": "<base64-encoded barcode>"}
```

Поддерживаемые команды: `auth` (авторизация), `scan` (передача кода).

## Сборка из исходников

### Зависимости

Установите через [vcpkg](https://github.com/microsoft/vcpkg) (триплет `x64-windows-static`):

```bash
vcpkg install boost-system:x64-windows-static
vcpkg install boost-asio:x64-windows-static
vcpkg install boost-beast:x64-windows-static
vcpkg install boost-signals2:x64-windows-static
vcpkg install boost-property-tree:x64-windows-static
vcpkg install openssl:x64-windows-static
```

### Сборка

```bash
cmake -B build/win-x64 \
  -DCMAKE_TOOLCHAIN_FILE=<путь_к_vcpkg>/scripts/buildsystems/vcpkg.cmake \
  -DVCPKG_TARGET_TRIPLET=x64-windows-static \
  -DCMAKE_BUILD_TYPE=Release

cmake --build build/win-x64 --config Release
```

Результат: `build/win-x64/Release/PhoneBarcodeScanner_win_x64.exe`

### Версионирование

Номер сборки определяется автоматически как количество коммитов в репозитории Git. Формат версии: `MAJOR.MINOR.BUILD` (например, `0.1.25`).
